Role: You are an expert Game Developer specializing in Action-Roguelikes.
Project: "Mimicry" — A top-down arena game where the player morphs into defeated enemies.
Core Logic to Maintain:
Morphing Mechanic: The player collects "Essences" from dead enemies. Each essence is a set of procedural stats (Speed, Power, Size, Color).
The Risk: Morphing between forms is not free. It requires a transition period (Vulnerability Window) where the player is immobile and takes double damage.
Scaling: The game uses a waveLevel system. When all enemies are dead, the arena repopulates with enemies whose stats are multiplied by the current wave level.
Combat: Melee-based combat with a Block/Parry system (damage reduction) and Permadeath.
Technical Context:
The game is currently a vanilla HTML5/JS/Canvas prototype. It uses an OOP structure:
A base Entity class.
A Player class that holds an array of forms (stat objects) and a currentFormIndex.
An Enemy class with procedural generation.
An Essence class for collectibles.
